## 💡 Key Concepts

### Payment for Remaining Amount
When an invoice has advance payments:
- Invoice total: ₹100,000
- Advance received: ₹30,000
- Remaining amount: ₹70,000
- **Payment QR is for ₹70,000 (NOT ₹100,000)**

This ensures customers only pay what they actually owe.

### Payment History Tracking
Every payment is recorded:
```json
"paymentHistory": [
  {
    "amountReceived": 30000,
    "paymentDate": "2024-01-15",
    "paymentMethod": "Bank Transfer",
    "referenceNumber": "TXN123"
  },
  {
    "amountReceived": 70000,
    "paymentDate": "2024-01-20",
    "paymentMethod": "UPI",
    "referenceNumber": "UPI456"
  }
]
```

### Auto-Calculated Status
Based on payment history sum:
- Remaining > 0 and Received = 0 → "Unpaid"
- Remaining > 0 and Received > 0 → "Partially Paid"
- Remaining ≤ 0 → "Paid"
